Audit Summary (Aura Health Next.js app)

Top findings:
1) Auth/session security is brittle: middleware trusts simple cookie flags; admin has insecure defaults/fallbacks; plain-text admin password comparison exists in one path.
2) Data architecture is inconsistent: two appointment systems + two status vocabularies (Pending/Confirmed/Cancelled vs new/contacted/scheduled/closed), plus JSON fallback divergence risk.
3) Signup/login likely fail under Supabase RLS due to direct browser inserts into public.users.
4) Booking reliability risk: queue numbers generated by client count-then-insert; duplicates possible under concurrency; fallback Q-001 on errors.
5) Chat API lacks rate limiting and healthcare guardrails.
6) UX regressions: header overlap/top clipping, specialist image framing issues, contact encoding artifacts.
7) SEO is minimal: no per-page metadata strategy, no sitemap/robots/schema JSON-LD.
8) Accessibility gaps: toasts not aria-live, modal semantics/focus management missing.
9) Healthcare trust gaps: privacy/data-use/legal/emergency guidance absent.
10) Growth instrumentation gaps: no analytics/event funnel/UTM attribution.

Roadmap:
Quick wins (1-2w): auth/session hardening baseline, header/visual bug fixes, SEO basics, analytics events.
Medium (2-6w): atomic queue generation, status/domain unification, remove JSON fallback in prod, accessibility pass, trust/legal pages.
Major (6-12w): CRM lifecycle automation, RBAC + audit logs, content+schema authority strategy.
